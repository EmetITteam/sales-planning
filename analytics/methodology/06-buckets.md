# 06 · Бакети по сумі і кількості

## Дефолтні бакети по сумі за місяць (USD)

```
до 99
100-199
200-299
300-399
400-499
500-599
600-699
700-799
800-899
900-999
1000+
```

SQL:
```sql
CASE
  WHEN monthly_sum < 100   THEN 'до 99'
  WHEN monthly_sum < 200   THEN '100–199'
  WHEN monthly_sum < 300   THEN '200–299'
  WHEN monthly_sum < 400   THEN '300–399'
  WHEN monthly_sum < 500   THEN '400–499'
  WHEN monthly_sum < 600   THEN '500–599'
  WHEN monthly_sum < 700   THEN '600–699'
  WHEN monthly_sum < 800   THEN '700–799'
  WHEN monthly_sum < 900   THEN '800–899'
  WHEN monthly_sum < 1000  THEN '900–999'
  ELSE '1000+'
END AS sum_bucket
```

Де `monthly_sum` — сума всіх покупок клієнта в межах календарного місяця:
```sql
SELECT
  client_code,
  DATE_TRUNC('month', sale_date) AS ym,
  SUM(sum_usd) AS monthly_sum
FROM sales
WHERE [золотий фільтр]
GROUP BY client_code, DATE_TRUNC('month', sale_date);
```

## Дефолтні бакети по кількості упаковок за місяць

```
1, 2, 3, 4, 5, 6, 7+
```

SQL:
```sql
CASE
  WHEN monthly_qty >= 7 THEN '7+'
  ELSE monthly_qty::text
END AS qty_bucket
```

## Логіка "клієнто-місяць"

- Один рядок результату = один клієнт у одному місяці
- Якщо клієнт купував у 3 місяцях — він буде у 3 рядках (різні бакети можуть бути)
- Сумарна цифра "Σ клієнто-місяців" ≠ "Σ унік. клієнтів" (унік. — UNION кодів)

## Альтернативні бакети — якщо користувач просить інші границі

Дефолтні границі (по 100$) — для більшості випадків. Користувач може попросити:
- **По 200$:** до 199 / 200-399 / 400-599 / 600-799 / 800-999 / 1000+
- **По 50$:** до 49 / 50-99 / 100-149 / ...
- **PETARAN-стиль:** 3-6 фл / 7-10 фл / 11+ фл

Запитуй у користувача якщо є сумнів. **Не вигадуй**.

## Бакети для упаковок — як рахується "кількість"

- Сума `qty` (поле "Количество (в ед. хранения)" з 1С) за всі рядки клієнта у місяці
- Один документ може містити різні позиції бренду — сумуємо всі qty по позиціях
- "Один документ з 3 уп" і "Два документи по 1 і 2 уп" — обидва дають qty=3

## "Стабільний за 4 місяців" (для PETARAN-стилю)

Для категоризації типу "Фокус 3-5":
- Беремо клієнта і всі його місяці у періоді
- Дивимось у скількох з них його qty потрапляє в діапазон (наприклад 3-6)
- Якщо в ≥4 — кваліфікується
- Місяці поза діапазоном НЕ дискваліфікують, просто не рахуються

```sql
WITH client_months AS (
  SELECT
    client_code,
    DATE_TRUNC('month', sale_date) AS ym,
    SUM(qty) AS monthly_qty
  FROM sales
  WHERE brand = 'PETARAN' AND [золотий фільтр]
  GROUP BY 1, 2
)
SELECT client_code
FROM client_months
GROUP BY client_code
HAVING COUNT(*) FILTER (WHERE monthly_qty BETWEEN 3 AND 6) >= 4;
```

## Why
Користувач явно попросила бакети по 100$ після кількох ітерацій з ширшими границями: "случайно, давай так, ДО 99, ПОТОМ 100 - 199, И ТАК ДАЛЬШЕ до 1000+". Це дефолт.

## How to apply
Якщо користувач каже "розподіл по чеках" або "сегментація по сумі" без уточнень — бакети по 100$. Якщо "по кількості упаковок" — 1-7+.
