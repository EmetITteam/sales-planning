# 04 · Когорти за recency (Активні / Сплячі / Втрачені)

## Канонічні границі

```
Активний:  recency ≤ 90 днів
Сплячий:   91 ≤ recency ≤ 180 днів
Втрачений: recency > 180 днів
```

Де `recency = REF_DATE - last_purchase_date` (у днях).

## REF_DATE — критично важливо

**REF_DATE = день після останнього дня даних**, НЕ `today()`, НЕ `NOW()`.

```sql
-- Правильно
SELECT MAX(sale_date)::date + INTERVAL '1 day' AS ref_date FROM sales;
-- Поточне: 2026-05-26 (бо останній день даних 2026-05-25)
```

**Чому не today():**
- Дані можуть бути не повні до сьогодні (sync вночі, є lag)
- Користувачка явно сказала "категорію рахуємо на 1.05, не на сьогодні" (PETARAN-кейс)
- Якщо REF_DATE > останнього дня даних — частина клієнтів несправедливо потрапить у "сплячих" просто бо ми не маємо їх свіжих покупок

## Розрахунок (SQL шаблон)

```sql
WITH last_purchase AS (
  SELECT
    client_code,
    MAX(sale_date)::date AS last_date,
    (SELECT MAX(sale_date)::date FROM sales) + 1 AS ref_date
  FROM sales
  WHERE [золотий фільтр]
    AND [бренд-фільтр якщо потрібно]
    AND [segment якщо потрібно]
  GROUP BY client_code
)
SELECT
  client_code,
  last_date,
  (ref_date - last_date) AS recency_days,
  CASE
    WHEN ref_date - last_date <= 90  THEN 'Активний'
    WHEN ref_date - last_date <= 180 THEN 'Сплячий'
    ELSE 'Втрачений'
  END AS cohort
FROM last_purchase;
```

## Розширена логіка PETARAN — "Фокуси"

Для деяких бренд-аналізів використовується інший набір сегментів:

| Фокус | Хто | Умова |
|---|---|---|
| Фокус 1 | Активний | recency ≤ 90 |
| Фокус 2 | Сплячий | 91-180 |
| Фокус 3 | Активний + стабільний середній | recency ≤ 90 AND ≥4 місяців у 3-6 фл/міс |
| Фокус 4 | Активний + стабільний високий | recency ≤ 90 AND ≥4 місяців у 7-10 фл/міс |
| Фокус 5 | Активний + топ | recency ≤ 90 AND ≥4 місяців у 11+ фл/міс |
| Фокус 6 | Сплячий+Втрачений з ≥2 покупок | recency >90 AND COUNT(DISTINCT doc_id) ≥ 2 |

**Класифікація 3-5 за найвищим обсягом** (клієнт у Фокусі 5 не дублюється у 3-4):
```
codes_5 = active ∩ (≥4 міс 11+)
codes_4 = (active ∩ (≥4 міс 7-10))   - codes_5
codes_3 = (active ∩ (≥4 міс 3-6))    - codes_5 - codes_4
```

## "Покупка" — що це

- **Покупка** = унікальний `doc_id` з префіксом `Реализация товаров и услуг ЗИН`
- НЕ дата, не місяць — саме документ
- Один документ може містити кілька позицій (рядків), але це одна "покупка" з точки зору клієнта

## Why
Користувач явно встановила ці границі при роботі по PETARAN: "категорію рахуємо на 1.05.2026" + "сплячий = 91-180 днів". Логіка зберігається для всіх брендів за замовчуванням.

## How to apply
Якщо користувач питає "скільки сплячих" без уточнень — рахувати по 91-180. Якщо хоче інші границі (наприклад 60-120, бо швидкий товар) — питати або застосовувати запитану логіку явно.
