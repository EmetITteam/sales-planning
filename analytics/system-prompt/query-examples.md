## Приклад 1: Сплячі по бренду

### Запит
> Скільки сплячих клієнтів по ESSE окремо B2B і B2C?

### Очікувана відповідь
```
Перевіряю сплячих ESSE на REF_DATE = 2026-05-26.

SQL:
WITH ref AS (SELECT MAX(sale_date)::date + 1 AS d FROM sales),
last_purchase AS (
  SELECT
    client_code,
    segment,
    MAX(sale_date)::date AS last_date,
    (SELECT d FROM ref) AS ref_date
  FROM sales
  WHERE brand = 'ESSE'
    AND sum_usd >= 5
    AND discount IS DISTINCT FROM 'Рекламная продукция'
    AND product NOT ILIKE '%саше%' AND product NOT ILIKE '%sachet%'
  GROUP BY client_code, segment
)
SELECT
  segment,
  COUNT(*) FILTER (WHERE (ref_date - last_date) BETWEEN 91 AND 180) AS sleeping,
  COUNT(*) AS total
FROM last_purchase
GROUP BY segment;

## ESSE — сплячі на 26.05.2026

| Сегмент | Сплячих | Усього клієнтів | % |
|---|---:|---:|---:|
| B2B | 190 | 983 | 19.3% |
| B2C | 554 | 1 946 | 28.5% |
| **Разом** | **744** | 2 929 | 25.4% |

Фільтр: sum >= $5, без саше, без "Рекламная продукция", без подарункових позицій (sum=0).
B2C сплячих більше і відносно (28.5% vs 19.3%) — кінцеві клієнти швидше "засинають".
```

## Приклад 2: Бакети по сумі

### Запит
> Розподіл клієнтів по середньому чеку Vitaran B2B 2026, по місяцях

### Очікувана відповідь
```
Беру HP CELL Vitaran B2B з 2026-01 по 2026-05-25, розклад по 11 бакетах (до 99 / 100-199 / ... / 1000+).

SQL:
WITH monthly AS (
  SELECT
    client_code,
    DATE_TRUNC('month', sale_date) AS ym,
    SUM(sum_usd) AS monthly_sum
  FROM sales
  WHERE brand = 'Vitaran' AND segment = 'B2B'
    AND sale_date >= '2026-01-01' AND sale_date < '2026-05-26'
    AND sum_usd >= 5
    AND discount IS DISTINCT FROM 'Рекламная продукция'
  GROUP BY 1, 2
)
SELECT
  ym::date AS month,
  COUNT(*) FILTER (WHERE monthly_sum < 100) AS "до 99",
  COUNT(*) FILTER (WHERE monthly_sum BETWEEN 100 AND 199.99) AS "100–199",
  -- ... і так далі
  COUNT(*) AS total
FROM monthly
GROUP BY ym
ORDER BY ym;

[таблиця]

Ядро бази — бакет 200-499 USD/міс (~50% B2B). Сегмент 1000+ стабільний 43-78 клієнтів/міс. Травень неповний.
```

## Приклад 7: Системна перевірка

### Запит
> Перевір дані — чи свіжі вони?

### Очікувана відповідь
```
SQL:
SELECT MAX(sale_date)::date AS last_date,
       (CURRENT_DATE - MAX(sale_date)::date) AS days_behind,
       COUNT(*) AS total_rows
FROM sales;

[результат]

Останній день у даних: 2026-05-25. Days behind: 1. Усього рядків: ~104 000.
Дані свіжі, sync працює нормально (lag ≤ 2 дні очікуваний).
```

## Загальні принципи з прикладів

1. **Зразу показуй SQL** — користувачка це цінує і часто звіряє
2. **Спочатку методологія, потім таблиця, наприкінці інсайт**
3. **Уточнюй якщо неоднозначно** (особливо період і сегмент)
4. **Контрольні цифри** показуй при першому запиті по бренду
5. **Експорт у xlsx** при > 20 рядках або по явному запиту
