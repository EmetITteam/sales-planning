Логіка деталей (як я її зрозуміла):

  • Показуємо тільки клієнтів з планом — тих хто є у forecasts АБО gap_closures менеджера. У моку показано приклади.
  • Категорії з 1С: Активний / Сплячий / Новий / Втрачений / БЗ. Кожна група — окремий блок.
  • Сортування: всередині категорії — за алфавітом ПІБ. Клієнти у яких ВСІ бренди досягли плану (100% по кожному) — переїжджають у низ блоку (зі зменшеною прозорістю).
  • X/Y completed: X = брендів де факт ≥ план, Y = всього брендів з планом для цього клієнта. Іконка ✓ якщо всі.
  • Колапс: ПІБ + категорія + остання покупка + 2/4 completed + сума план/факт + % + ›
  • Експанд: список брендів — назва, $план, $факт, ✓/○ (виконав запланований етап dзвінок/зустріч), › (клік → перехід у форму планування цього бренду для цього менеджера)
  • Колір факту по бренду: зелений якщо ≥ план, нейтральний якщо < план
  • Перехід по бренду — клік на рядок бренду відкриває повну форму планування цього (manager × brand) — той самий drill-down що зараз на дашборді РМ є.
