## 🎯 Track A · Admin «Уся компанія» дашборд

### Що це

Окрема read-only вкладка для admin (потім — за toggle для обраних юзерів) яка показує план/факт по всій компанії включно з не-планувальними підрозділами (Колл-центр, Лазерхауз, Адасса, Чугуй, Хайленко).

**НЕ** містить:
- Клієнтського планування (без блоків forecast/gap)
- Можливості редагування (read-only)
- Дрилу до клієнтів

### Архітектура UI

**Затверджена структура** (з preview `/admin-overview-preview-v2.html`):

| Секція | Що показує |
|---|---|
| **Hero (4 cards)** | План компанії · Факт · % виконання · Кількість без факту |
| **Фільтри (chip-и)** | Період (місяць/квартал/рік) + Група підрозділів |

### Access control

**Як працює:**
1. Колонка `users.can_view_company_overview boolean DEFAULT false` (нова міграція M10)
2. Endpoint `/api/admin/dashboard-permissions` (GET+POST, admin only) — список юзерів + toggle
3. Admin page `/admin/dashboard-permissions` — таблиця з пошуком + checkbox (як `/admin/stage-edit-permissions`)
4. У `/api/auth/me` додаємо `canViewCompanyOverview: boolean` fetch з БД (як ми робимо з `canEditStagesAfterFinalize`)
5. Frontend: пункт меню «Уся компанія» з'являється у nav коли `user.canViewCompanyOverview === true` АБО `role === 'admin'`
6. Backend гард: `/admin/company-overview` маршрут 403 якщо нема дозволу

**Дефолт:** тільки admin бачить. Користувач може у адмінці поставити галочку конкретним юзерам (наприклад, директору продажу).

## 📡 Track C · 1С Action 5 — розширення

### Проблема

`getRegionData` не повертає факт для 4 підрозділів: Колл-центр, Адасса, Полтава\* (Чугуй), Чернівці\* (Хайленко).

Зараз 1С фільтрує по якомусь критерію «active» — ми його не знаємо.

### Що просимо у Андрія

Розширити Action 5 одним з двох варіантів (Андрій обере що зручніше):

**Варіант 1: прапор у запиті**
```json
{
  "action": "getRegionData",
  "payload": { "login": "sdu@emet.in.ua", "period": "2026-05", "includeAll": true }
}
```
Тоді при `includeAll=true` повертає всі 13 підрозділів з фактом.

**Варіант 2: новий action**
```json
{ "action": "getCompanyData", "payload": { "period": "2026-05" } }
```
Окремий метод без фільтру по active. Має сенс якщо `getRegionData` має складну логіку прав для РМ і не варто її чіпати.

### Очікувана відповідь

Така ж структура як зараз `GetRegionDataResponse` (`regions[]`), просто з більшою кількістю елементів. Адаптер у `onec-adapters.ts` лише прибере фільтр `isActiveDivision` для нового endpoint.

### Як спілкуватись з Андрієм

Підготувати чітку специфікацію (в одному месенджу йому):
1. Контекст — для якої сторінки
2. Бажаний JSON request/response
3. Тестовий запит
4. Що буде якщо не зробить — поки н/д на сторінці, потім зробимо

### Зусилля

- Андрій: ~1 день роботи + тестування
- Ми (інтеграція): ~2 години після того як зробить
