# Стратегічний KPI-дашборд («Стратегія»)

**Маршрут:** `/admin/strategic-kpi` · **Доступ:** `itd@emet.in.ua` (admin) + `sdu@emet.in.ua` (директор продажів) — див. `isStrategicKpiLogin`.
**У навігації:** вкладка «Стратегія» у верхній панелі поряд з «Планування» / «Огляд компанії».
**Статус:** у проді з 2026-07-03 (ETALON `etalon-strategic-kpi-2026-07-03`).

Окремий продуктовий погляд на виконання річної/місячної стратегії по 11 брендах у розрізі 3 каналів (представництва / колл-центр / дистриб'ютори). НЕ дублює «Планування» — фокус на стратег-цілях (`strategic_targets`) + грошовому % виконання.

---

## 1. Джерело істини для % виконання плану

**Грошовий % (hero + per-channel) береться з «Огляд компанії»** (`/api/admin/company-overview`) — той самий 1С Action 4 (план) + Action 5 (факт), що бачить директор в Огляді та менеджер у Плануванні. **Дашборд НЕ робить власних викликів у 1С за планом/фактом.**

- Хук `useCompanyOverviewExec(period, enabled)` тягне Огляд і згортає `divisions[].segments[segCode]` у мапу `brand × channel → {план, факт}`.
- Мапінг: `groupKey → channel` (representations→representatives, call-center→call_center, distributor-*→distributors), `segCode → brand` (`VITARAN→Vitaran`, …, `IUSE→IUSE`, `BAD→БАД`).
- Тільки місячний період (Огляд приймає `YYYY-MM`). Для квартал/півріччя/рік мапи немає → відкат на **клієнтський %** (buyers_monthly).

**Чому так (ADR-19):** раніше дашборд рахував план/факт власними викликами 1С — вони таймаутили, % відкочувався на клієнтський (Vitaran показував 90.9%, Ellanse 107.4% замість грошових). Єдине джерело усунуло розбіжності.

## 2. Які канали рахуються у бренд

Канал входить у % бренду **лише якщо має стратег-тригери** (`isChannelActive`):

| Канал | Активний для |
|---|---|
| Представництва | усі бренди |
| Колл-центр | ESSE, IUSE Coll., БАД |
| Дистриб'ютори | лише Ellanse |

Приклад: у Vitaran є дистриб'юторський план/факт у 1С, але для Vitaran дистри не активні → **у % не входять** (щоб hero збігався з блоком «Представництва»). Сума з дистрами враховується лише там, де є тригери (Ellanse).

## 3. Hero-баннер

- Великий `% виконання плану` = Σ факт / Σ план по активних каналах бренду.
- Кольори: **≥100 зелений · 60–99 жовтий · <60 червоний**.
- Коли бренд охоплює **≥2 канали** — під великим % дрібним шрифтом розбивка по підрозділах (`Представництва X% · Колл-центр Y%`), щоб було видно склад загального %.
- Категорії клієнтів бренду у періоді: Нові / Активні / Сплячі / Втрачені (компонент `HeroCategories`).

## 4. Канальні блоки

Для кожного активного каналу — блок з 4 KPI-картками (унікальні за рік, купують у міс., ср/уп на клієнта, середній чек) з таргетами `strategic_targets` + прогноз. Плюс розкладка категорій клієнтів у цьому каналі (RPC `get_brand_channel_categories`). Дистриб'ютори (Ellanse) — тільки кількість семінарів, грошових метрик у 1С немає.

## 5. Сегмент IUSE (SB / hair / Coll.)

IUSE у 1С — **один сегмент** (єдиний грошовий план, без розшивки по підбрендах). На дашборді:
- Hero — грошовий % по всьому сегменту (з Огляду, ключ `IUSE`).
- Всередині — **3 повні блоки підбрендів** (SB / hair / Coll.), кожен зі своїми KPI-картками, промо і **власною розбивкою клієнтів по категоріях** (`sub_brand_channel_categories`, per суб-бренд×канал).
- На блоках підбрендів **% виконання НЕ показуємо** — плану на підбренд у 1С немає, брати ніде. % є лише на hero сегмента.
- КЦ у сегменті — тільки в IUSE Coll.

## 6. Інші блоки

- **Акції** (окремий toggle у ряді брендів): промо-розбивка + реактивація (класифікація на 1-ше число місяця, % по кількості клієнтів).
- **Ellanse-специфіка:** «Впервые обучені» у представництвах (авто-derive із продажів по (семінар, підрозділ), фільтр по ELLANSE у назві), річний план vs факт семінарів.
