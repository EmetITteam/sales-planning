# 📡 Action 5 — нічого розширювати не треба

## Контекст (виправлено 2026-05-25)

Я спочатку думав що Action 5 фільтрує по архівним/не-архівним підрозділам — це **було неправильно**.

**Реально:** `getRegionData` фільтрує по менеджерам — якщо у підрозділі нема жодного менеджера сконфігурованого у 1С, він не потрапляє у відповідь. Підрозділи Колл-центр / Адасса / Чугуй / Хайленко зараз не повертаються бо за ними не закріплені менеджери у потрібному регістрі.

## Що робить користувач

Конфігурує менеджерів у 1С за відповідними підрозділами. Після цього Action 5 автоматично почне повертати їх — без зміни API.

## Що робимо ми

Нічого. Поточний код буде працювати як тільки 1С почне віддавати дані. Адаптер не треба міняти.

## Як перевірити коли все буде готово

```bash
node scripts/diag-divisions.mjs
```

Має показати всі 13 підрозділів у Action 5 секції (зараз тільки 11 — без Колл-центру, Адасси, Полтави, Чернівців).
