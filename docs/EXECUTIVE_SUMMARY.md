# Sales-Planning → EMET CRM
## Опис проекту розширення

**Версія:** 8.0 · **Дата:** 2026-06-02
**Підготувала:** IT Director EMET

---

## 1. Для кого і навіщо

**Користувачі:** менеджери відділу продажу + керівництво відділу.

**Навіщо:** винести щоденну роботу менеджера з 1С у мобільну CRM, щоб працювати з телефона у полі. 1С залишається базою даних.

Це **стратегічна зміна інструменту** відділу продажу — не точкове покращення. Поточна система перетворюється з модуля планування у повноцінну CRM, яка покриває весь робочий день менеджера від зустрічі до замовлення і рекламації.

## 2. Що працює зараз технічно

### Поточні модулі у експлуатації з травня 2026

| Модуль | Аудиторія | Технічне покриття |
|---|---|---|
| Планування продажів | Менеджери + керівництво | Місячне планування цілей по 9 ТМ з прив'язкою до клієнтів, finalize-механіка, window-lock періодів планування, історія планів |
| Мої клієнти (CRM-база) | Менеджери | Клієнтська база з кешуванням, фільтрами і деталізацією, drill-down картка клієнта з історією зустрічей, статусом активності, прогнозами |
| Огляд компанії | Керівництво | Виконання плану по регіонах + по брендах, агрегати продажів, аналітика «відстають від плану», структура продажів |

### Технологічна основа поточної системи

**Технічно:**
- Веб-фреймворк: **Next.js 16** з App Router і React Server Components
- UI-бібліотека: **React 19** + TypeScript 5 для типобезпеки
- Стилі і компоненти: **Tailwind CSS 4** + **shadcn/ui** (Radix-primitives)
- Стан і дані: **Zustand** (глобальний store) + **SWR** (cache + stale-while-revalidate)
- База даних: **Supabase Postgres** з підготовленими RLS-політиками
- Авторизація: **JWT cookies з ECDSA-підписами** (HttpOnly, secure)
- Хостинг: **Vercel** з автоматичним розгортанням master + preview-середовища для feature-branches
- Інтеграція з 1С: **13 HTTP-методів** через захищений proxy-шар `/api/onec`
- Тестування: **240 одиничних тестів** (Node-нативний test runner) + перевірка архітектурних інваріантів перед кожним оновленням
- Шрифти: Plus Jakarta Sans (текст) + JetBrains Mono (числа і час)

**Простою мовою:**

Система побудована на сучасному веб-стеку, що використовується більшістю продуктових компаній (Next.js + React). Працює як веб-додаток у браузері — без встановлення на пристрій, відкривається з телефона, планшета, ноутбука однаково. База даних розміщена у хмарі (Supabase), з резервним копіюванням і можливістю відкату до конкретного моменту у часі при переході на платний тариф. Авторизація захищена сучасними криптографічними підписами. Усі звернення до 1С проходять через захищений шар, який пропускає лише авторизовані запити. Кожне оновлення коду автоматично запускає 240 тестів + перевірку архітектурних правил перед тим, як код потрапляє у систему — це знижує ризик помилок при додаванні нових модулів.

### Зрілість проекту

- ~57 000 рядків коду
- 13 інтеграцій з 1С працюють стабільно з січня 2026
- Адаптивний мобільний інтерфейс протестовано
- Архітектурні правила фіксуються автоматично — не може бути додано нову залежність чи зламано існуючу інтеграцію без явного оновлення правил
- Bug-tracking через GitHub Issues, історія всіх змін у git

## 3. Що додаємо у складі CRM-розширення

### Перенесення з існуючої СРМ Встречі

**Зустрічі** — функціонал зараз живе у окремому додатку (meeting-4.0). Переносимо у нашу CRM і об'єднуємо з рештою CRM-функціоналу. Покращуємо: стабільніша геолокація, мобільна форма створення і редагування, фонова синхронізація з 1С через буферну чергу, синхронізація з Google Calendar з механізмом повторних спроб.

### Нові модулі (розробляємо з нуля)

- **Деталізація продажів** — рядкова історія покупок клієнта (з січня 2025) одним тапом на картці клієнта; для керівництва — автоматичні аналітичні віджети (виконання плану, найкращі менеджери по ТМ за конкурсом, дебіторка по регіонах)
- **Дебіторка** — список і деталізація заборгованості клієнтів, віджет на картці з показниками простроку (30/60/90/90+ днів)
- **Рекламації** — нова структурована секція CRM з прив'язкою до клієнта замість поточного загального Telegram-каналу. Поточна база рекламацій у Bitrix24 не ламається — додаємо інтерфейс над нею
- **Замовлення / Реалізації** — створення з мобільного, з кошиком, вибором контракту, перевіркою наявності, gift-логікою, статусним workflow «чернетка → проведено → реалізація»

### Нові інтеграції і компоненти

- 16 нових методів 1С (специфікація з прикладами запитів/відповідей готова)
- Окремий sync-worker для відправки даних у 1С батчами (буферна архітектура, щоб UI не очікував 1С)
- Окремий sync-worker для Google Calendar з retry + dead-letter queue
- Окремий sync-worker для нічної синхронізації деталізації продажів
- Інтеграційний шар з Bitrix24 для блоку рекламацій (через існуючий reclamation-app)
- Підключення Sentry для моніторингу помилок з session replay
- Перехід на Supabase Pro для PITR і збільшення ліміту бази

## 5. Терміни і години роботи

### Розподіл по блоках

| Блок | Зміст | Frontend години | Календарно |
|---|---|---|---|
| Підготовка | Документація, специфікації, дизайн | ✅ Готова | — |
| Зустрічі | Перенесення з meeting-4.0 + покращення + буферна синхронізація + інтеграція Google Calendar | 120-160 годин | 3-4 тижні |
| Деталізація продажів | Бекфіл з 2025 + щоденна синхронізація + аналітичні віджети + Best Manager | 56-72 годин (паралельно з Зустрічами) | 5-6 тижнів кумулятивно |
| Дебіторка | Інтерфейс на картці + standalone view + кеш з TTL | 40-56 годин | 7-8 тижнів кумулятивно |
| Рекламації | Інтеграція з reclamation-app + counter-widget | 24-32 годин | 8 тижнів кумулятивно |
| Замовлення / Реалізації | Форма з кошиком + multi-currency + gift-логіка + 9 нових методів інтеграції | 120-160 годин | 12-13 тижнів кумулятивно |

**Загальний обсяг розробки інтерфейсу:** 360-480 годин (~10-13 робочих тижнів повної зайнятості).

**1С-розробка:** ~50% часу 1С-розробника протягом усього проекту, розподілено по чергах по блоках. Конкретний час залежить від наявних можливостей 1С-розробника і узгодженої черги.

### Критична залежність

Готовність нових 1С-методів — головна змінна. Якщо 1С-черга затримується, етапи 1.5 / 2A / 3 зрушуються пропорційно (frontend готовий, але не може стартувати).

## 8. Що отримаємо на виході

### Результат для менеджера

- Менеджер виконує всю щоденну роботу у CRM з телефона або ноутбука у будь-якому місці
- Замовлення оформлюється прямо у клієнта на зустрічі, не повертаючись у офіс
- Історія покупок клієнта доступна одним тапом під час розмови
- Дебіторка клієнта видно на його картці одразу
- Рекламації ведуться у структурованій секції CRM замість Telegram-каналу
- Зустрічі фіксуються з геолокацією без зайвих кроків
- Доступ до 1С менеджеру більше не потрібен у щоденній роботі

### Результат для керівництва

- Живі аналітичні віджети у системі замість ручних Excel-вивантажень
- Автоматичний розрахунок переможців конкурсу «Найкращий менеджер по ТМ» за діючими умовами, з виключенням закупок спікерів і фільтрами по виконанню планів
- Деталізація продажів по клієнтах — для звітів і прийняття рішень
- Огляд дебіторки і її aging — без походів у 1С

### Результат для системи

- 6 робочих модулів у CRM (3 існуючих + 4 нових/перенесених)
- Захист фінансово-чутливих даних через політики доступу і резервне копіювання
- Моніторинг помилок у роботі через Sentry
- Документація проекту з реєстром рішень фіксує контекст для подальшого розвитку
- Архітектура з буферною синхронізацією масштабується для додавання нових модулів без переробки
