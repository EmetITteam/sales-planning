# Sales-Planning → EMET CRM
## Опис проекту розширення

**Версія:** 7.0 · **Дата:** 2026-06-02
**Підготувала:** IT Director EMET

---

## 1. Для кого і навіщо

**Користувачі:** 21 менеджер відділу продажу + керівництво відділу.

**Навіщо:** зараз менеджер всю щоденну роботу виконує у 1С, яка живе на офісному сервері. Це означає: щоб занести замовлення після зустрічі, переглянути історію клієнта чи перевірити дебіторку — потрібен ноутбук у офісі. Розширюємо існуючу систему `sales-planning` так, щоб менеджер виконував усю цю роботу з телефона чи планшета у будь-якому місці — у машині між візитами, на зустрічі у клієнта, з дому. 1С залишається як база даних компанії, але прямого доступу менеджера до неї більше не потрібно.

## 2. Що буде у складі продукту

### Існуючі модулі (вже у роботі з травня 2026)

- **Планування продажів** — менеджери планують місячні цілі по 9 ТМ, керівництво бачить заповнюваність
- **Мої клієнти** — клієнтська база менеджера з фільтрами і деталізацією
- **Огляд компанії** — виконання плану по регіонах і брендах для керівництва

### Нові модулі (додаємо)

- **Зустрічі** — фіксація зустрічей з геолокацією, з мобільного у полі
- **Деталізація продажів** — рядкова історія покупок клієнта (з 2025) одним тапом на картці клієнта; для керівництва — автоматичні аналітичні віджети (виконання плану, найкращі менеджери по ТМ за конкурсом, дебіторка по регіонах)
- **Дебіторка** — список і деталізація заборгованості клієнтів, віджет на картці
- **Рекламації** — нова секція CRM з прив'язкою до клієнта замість поточного загального Telegram-каналу
- **Замовлення / Реалізації** — створення з мобільного, з кошиком, вибором контракту, перевіркою наявності, gift-логікою

### Інтеграції

- 1С — для всіх бізнес-даних (продажі, клієнти, дебіторка, замовлення)
- Google Calendar — для синхронізації зустрічей у календар менеджера
- Bitrix24 + Telegram-бот — як існуюча база для блоку рекламацій (додаємо UI у CRM, наявну систему не ламаємо)
- Sentry — моніторинг помилок системи у роботі

## 3. Що вже зроблено

### Працююча система

- 3 модулі у роботі з травня 2026, щодня використовуються 21 менеджером
- 13 методів інтеграції з 1С працюють стабільно з січня 2026
- 240 автоматичних тестів + перевірка цілісності перед кожним оновленням
- Адаптивний інтерфейс для телефонів і планшетів

### Підготовча документація

- Технічний план проекту з декомпозицією по блоках, оцінками, ризиками — 1100+ рядків
- Специфікація для 1С-розробника на 16 нових методів з прикладами запитів/відповідей, виключеннями, вимогами до швидкодії — 1050 рядків
- Реєстр з 17 архітектурних рішень з контекстом, обґрунтуванням і наслідками
- Аудити трьох вихідних систем: поточного додатка зустрічей, недороблена попередня версія з замовленнями, чинної системи рекламацій
- Специфікація алгоритму найкращого менеджера по ТМ за діючими конкурсними умовами
- Детальна специфікація блоку деталізації продажів зі схемою бази даних

### Дизайн

- Дашборд зустрічей у нашому glass-стилі (3 варіанти розглянуто, фінальний обрано)
- Форма зустрічі (2 варіанти розглянуто, фінальний обрано)
- Мобільна адаптивність протестована, кнопки розраховані на тач-екран

## 7. Що отримаємо на виході

### Результат для менеджера

- Менеджер виконує всю щоденну роботу у CRM з телефона або ноутбука у будь-якому місці
- Замовлення оформлюється прямо у клієнта на зустрічі, не повертаючись у офіс
- Історія покупок клієнта доступна одним тапом під час розмови
- Дебіторка клієнта видно на його картці одразу
- Рекламації ведуться у структурованій секції CRM замість Telegram-каналу
- Зустрічі фіксуються з геолокацією без зайвих кроків
- Доступ до 1С менеджеру більше не потрібен у щоденній роботі

### Результат для керівництва

- Живі аналітичні віджети у системі замість ручних Excel-вивантажень
- Автоматичний розрахунок переможців конкурсу «Найкращий менеджер по ТМ» за діючими умовами, з виключенням закупок спікерів і фільтрами по виконанню планів
- Деталізація продажів по клієнтах — для звітів і прийняття рішень
- Огляд дебіторки і її aging — без походів у 1С

### Результат для системи

- 6 робочих модулів у CRM (3 існуючих + 3 нових)
- Захист фінансово-чутливих даних через політики доступу і резервне копіювання
- Моніторинг помилок у роботі через Sentry
- Документація проекту з реєстром рішень фіксує контекст для подальшого розвитку
