# Звіт РОП → CSO/CMO (Лист 4) — карта даних для звірки

Мапінг макета `docs/mockups/rop-report.html` на реальні джерела даних: що вже
рахується, що треба агрегувати, що вводиться вручну, що створювати з нуля.

**Легенда статусів:**
- ✅ **ГОТОВО** — уже рахується в системі, беремо напряму
- 🔧 **АГРЕГАЦІЯ** — дані є, треба згорнути (бренд→регіон або регіон→компанія)
- ✍️ **РУЧНИЙ** — вводить РМ/РОП (поле вже є в `weekly_report_notes`)
- 🆕 **НОВЕ** — немає ні даних, ні поля, треба створювати

## 4.1 Зведена таблиця (по рядку на регіон)
| Колонка | Джерело | Статус | Чим закриваємо |
|---|---|---|---|
| Регіон + к-сть представництв | getRegionData `regionName` | ✅ | напряму |
| % на дату | getRegionData fact/plan (`pctOf`) | ✅ | як `brandRows` у звіті, але на весь регіон |
| Мітка (В ПЛАНІ/РИЗИК/ВІДСТАВАННЯ) | `statusBadge(forecastPct)` | ✅ | функція є |
| Червоні бренди (чипи) | per-brand `forecastPct < 80` у цьому регіоні | 🔧 | порахувати мітку по кожному бренду, вивести «червоні» |
| Причина за стандартом | `weekly_report_notes.reason` (**per бренд**) | ✍️🔧 | текст РМ уже вводить; **потрібне правило** який бренд показати на рівні регіону (червоний/гірший) |
| Дія на тиждень | `weekly_report_notes.action` (**per бренд**) | ✍️🔧 | те саме правило згортки бренд→регіон |
| Обіцянка→факт (Так/Ні) | `promise_check` (done + причина) | 🔧 | згорнути по регіону: всі виконані = Так, інакше Ні + причина |

## Ключові «дірки» для рішення (до реалізації)
1. **Правило згортки бренд→регіон** для причина/дія в таблиці 4.1 — показувати червоний/гірший бренд? чи кілька? *(потрібне рішення користувача)*
2. **«В термін» планування** — чи пишемо ми зараз **час** фіналізації плану? Якщо ні — додати, щоб порівнювати з дедлайном. *(перевірити в БД)*
3. **Ринкові сигнали (4.5)** — нова міні-фіча (поле + ввід РОП). Окремий крок.
4. **Multi-region агрегація** — зараз звіт вантажить **1 регіон**; для РОП потрібен серверний збір усіх 8 (новий API поверх існуючих розрахунків).

## Наступні листи (після Листа 4)
- **Лист 5 — Протокол SMA (Частина А):** рішення наради (ЩО+ХТО+КОЛИ+статус). Готує РОП. Правило §6.4.
- **Лист 6 — Записка SMA (Частина Б):** аналітика для Owner (Б1 зведена таблиця, Б2 червоні зони, Б3 реєстр обіцянок, Б4 підсумки планування, Б5 ринкові сигнали, Б6 висновок CSO). Готує CSO, тільки Owner+CSO.
- Позначені у файлі наради «потребує підтвердження Owner перед закріпленням як стандарт» → робимо ПІСЛЯ обкатки Листа 4.
